# 五子棋 AI 对战（C++ 移植版）

本项目将 [Touhou Little Maid](https://github.com/TartaricAcid/TouhouLittleMaid) 中的五子棋 AI 从 Java 移植到 C++。  
原始 AI 使用启发式评估、Alpha-Beta 剪枝、迭代加深以及 VCF/VCT 算杀等算法。移植后完全保留原有能力，并提供了一个简易的控制台对战程序，方便开发者集成或学习。

> 代码由 **DeepSeek AI** 自动生成，并经过人工验证与适配。

## 特性

- 15×15 标准五子棋棋盘
- 玩家执黑先手，AI 执白后手
- 支持 AI 难度自定义（搜索深度、候选点数量、算杀模式等）
- 使用 Zobrist 哈希缓存局面，提升搜索效率
- 启发式落子点排序（高/低优先级 + 杀棋点）
- Alpha-Beta 剪枝的 Minimax 搜索
- 迭代加深策略，动态调整搜索深度
- 可选 VCF（连续冲四取胜）和 VCT（活三 + 冲四组合取胜）算杀
- 调试模式可输出详细的搜索统计信息

## 编译与运行

### 环境要求

- C++11 或更高版本编译器（GCC、Clang、MSVC 均可）
- 仅依赖 C++ 标准库（无第三方依赖）

### 编译示例（Linux / macOS）

```bash
g++ -std=c++11 -O2 THMGomokuAI.cpp ZhiZhangAIService.cpp -o gomoku_ai
```

### 运行

```bash
./gomoku_ai
```

## 使用说明

启动后，程序会询问是否使用默认难度配置。默认配置为：搜索深度 = 6，最大候选点 = 10，算杀关闭。

若选择手动配置，则依次输入：
- 是否开启调试模式（Y/N）
- 搜索深度（正整数）
- 最大候选点数目（正整数）
- 算杀模式（0=关闭，1=VCF，2=VCT）
- 算杀深度（若算杀模式不为0）

游戏开始后，棋盘会以字符形式打印：
- `@` 表示黑棋（玩家）
- `O` 表示白棋（AI）
- `.` 表示空位

玩家输入坐标 `x y`（范围 0 ~ 14），例如：`7 7`。  
AI 自动落子并打印棋盘，直到分出胜负或平局。

## 核心算法简介

1. **棋型评估**  
   对每个空位，扫描四个方向（水平、垂直、两对角线）的棋子序列，与预定义的棋型模式（连五、活四、冲四、活三、眠三等）匹配并打分。AI 同时评估自己落子和对手落子的得分。

2. **启发式落子点筛选**  
   根据评估分数将落子点分为高优先级（关键攻防）、低优先级、候选点等，并限制每层搜索的最大节点数，大幅缩小搜索空间。

3. **Alpha-Beta 剪枝**  
   在 Minimax 搜索过程中维护 alpha（AI 最佳分数）和 beta（对手最佳分数），当 alpha >= beta 时剪去无用分支。

4. **迭代加深**  
   从浅层深度开始搜索，逐步加深，直到达到配置的最大深度或找到必胜解。配合局面缓存，可复用中间结果。

5. **算杀（VCF / VCT）**  
   在正式搜索前，先进行特定深度的杀棋搜索。VCF 仅考虑冲四，VCT 考虑冲四和活三组合，若找到必胜路径则直接落子，否则回退到常规搜索。

## 移植说明

- Java 原版使用了 `int[][]` 和 `List<Point>`，C++ 版本改用 `std::vector<std::vector<int>>` 和 `std::vector<Point>`。
- 随机数生成：Java 使用 `ThreadLocalRandom`，C++ 使用 `std::mt19937_64` + `std::uniform_int_distribution`。
- 哈希表缓存：Java 使用 `HashMap<Long, SituationCache>`，C++ 使用 `std::unordered_map<int64_t, SituationCache>`。
- 调试输出：Java 通过 SLF4J 日志，C++ 直接输出到 `std::cout`。
- 功能完全对齐，包括算杀、启发式、迭代加深等核心逻辑。
